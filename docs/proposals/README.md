# Semble Proposals

This directory holds proposals that are not yet accepted as roadmap items or
architecture decisions.

Use proposals for work that needs discussion before it becomes stable project
direction. That can include product, workflow, CLI, UX, or architecture-adjacent
changes.

## Lifecycle

- `draft`: under discussion
- `accepted`: ready to become implementation work, roadmap scope, or an ADR
- `rejected`: not moving forward
- `superseded`: replaced by a newer proposal

## When To Use An ADR Instead

Use `docs/adr/` when the question is about a long-lived architectural decision,
especially one that affects:

- the consumer-facing API
- composition semantics
- layering rules
- other durable structural boundaries
